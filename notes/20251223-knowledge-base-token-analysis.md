# 知识库系统 Token 消耗分析

## 增加的消耗（每次对话）

| 项目 | 估算 Token |
|------|-----------|
| Steering 规则加载 | ~300-500 |
| 读取 INDEX.md | ~200-400 |
| 读取相关解决方案文件 | ~500-1500 |
| 保存内容到知识库 | ~200-400 |
| **每次对话额外消耗** | **~1200-2800** |

## 节省的消耗（长期）

| 场景 | 节省 Token |
|------|-----------|
| 重复问题直接调取方案 | ~2000-5000 |
| 跨项目复用解决方案 | ~3000-8000 |
| 新设备不用重新探讨 | ~5000-15000 |

## 结论

- **短期**：每次对话多消耗 ~1500 Token（约 1-2%）
- **长期**：同一问题问 2 次以上即可回本，知识积累越多节省越明显

## 优化建议

1. 手动触发检索（而不是每次自动）
2. 精简 INDEX.md 只保留关键信息
3. 设置阈值，只检索高匹配度内容
